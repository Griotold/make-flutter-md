# 인앱 튜토리얼 (표준 기능)

## 왜 이 기능이 모든 신규 앱에 표준인가

앱 아이디어 출처가 Claude 또는 마켓 분석이므로 본인도 앱 사용법을 모를 수 있다. 본인 + 일반 사용자 모두를 위한 표준화된 온보딩이 없으면 앱을 처음 열었을 때 무엇을 해야 하는지 알 수 없다. 모든 신규 앱에 기본 탑재한다.

## UX 패턴 — Sudoku 힌트 스타일

- 3~4장 카드, Flutter `PageView`로 좌우 스와이프.
- 각 카드 = 비주얼(이미지 or 영상) + 텍스트 1~2줄.
- 페이지 인디케이터(점) 하단 표시. 첫 카드부터 마지막까지 자유 이동.
- 마지막 카드에 "시작하기" 버튼.

## 표시 시점

- **첫 실행 시 자동**: `app_settings` 박스의 `tutorial_seen` 플래그가 `false`이면 앱 시작 직후 표시.
- **재시청 진입점 필수**: 설정 화면 또는 메인 화면 ⓘ 버튼에서 언제든 재생. 1~2탭 내 접근 보장.

## 카드 비주얼 정책 (이미지 필수, 영상 선택)

- 모든 카드에 정적 이미지 필수. Nano Banana(Gemini 2.5 Pro image generation)가 생성.
- 영상은 **모션이 정보일 때만** 옵트인 — 제스처, 슬라이드, 애니메이션 타이밍 등 설명. 단순 "여기를 탭"류는 이미지로 충분.
- 기본값: 이미지(default-cheap). 영상이 필요한 카드만 빌드 설정에서 `motion: true` 명시.
- 이미지 일러스트는 **실제 앱 UI 레이아웃/색상/요소를 반영**. 가상 다른 앱 화면 사용 금지.
- 음성 없음. 영상이 있어도 무성 + 화면 위 텍스트로 충분.

## 모션 여부 결정 권한 (Claude 제안 + 사용자 승인)

튜토리얼 빌드 직전, Claude Code가 카드별로 모션 필요성을 판단하고 표로 제안한다.

| # | 텍스트 요약              | 제안   | 이유                          |
|---|--------------------------|--------|-------------------------------|
| 1 | "빈 셀을 탭하세요"        | image  | 정적 인디케이터로 충분         |
| 2 | "숫자를 드래그해 옮기기"  | video  | 드래그 모션이 핵심 정보        |

- 사용자가 표를 확인하고 카드별로 OK/변경 결정한 뒤에만 빌드 실행.
- 한 카드라도 미승인이면 빌드 스크립트가 거부.
- 변경 사항은 빌드 설정 파일에 기록 → 다음 빌드부터 같은 결정 재사용 (재확인 불필요).

## 빌드 타임 생성 파이프라인

런타임 생성 금지 (사용자 대기/네트워크/비용 문제). 모두 빌드 타임에 사전 생성한다.

1. `tool/tutorial.yaml`에 카드별 정의 작성:

   ```yaml
   cards:
     - text_key: tutorial.card1
       image_prompt: "..."
       motion: false
     - text_key: tutorial.card2
       image_prompt: "..."
       motion: true
       motion_prompt: "셀이 하이라이트되며 들어옴"
   ```

2. 빌드 스크립트 실행: `dart run tool/build_tutorial.dart`
   - 모션 결정 확인 (미승인 카드 있으면 중단)
   - 카드별로 Nano Banana API 호출 → 이미지 생성
   - `motion: true` 카드는 추가로 image-to-video API 호출 → 3~5초 mp4 생성

3. 출력 위치:
   - 정적 카드: `assets/tutorial/card_N.png`
   - 모션 카드: `assets/tutorial/card_N.mp4`

4. `GEMINI_API_KEY`는 `.env`에서 읽음. `.env`는 `.gitignore`에 포함 — 절대 커밋 금지.

## 재생 위젯 패턴

- Flutter `PageView` 사용.
- 각 페이지가 파일 확장자 보고 분기:
  - `.png` → `Image.asset(...)`
  - `.mp4` → `video_player` 패키지 (autoplay, looping, 컨트롤 숨김)
- 보이지 않는 페이지의 영상은 일시정지 (배터리/메모리 절약).
- `chewie` 불필요 — 컨트롤 UI 표시 안 함.

## 다국어

- 비주얼(이미지/영상)은 **언어 무관** — 한 번만 생성.
- 텍스트는 Flutter UI 레이어에서 ARB/intl로 렌더 (`text_key`로 참조).
- 비주얼을 언어별로 재생성하지 않으므로 빌드 비용 절감 효과 큼.

## Hive 플래그

- 박스 이름: `app_settings`
- 키: `tutorial_seen` (bool)
- 첫 실행 시 `false` → 튜토리얼 종료 후 `true`로 갱신.
- "다시 보기" 진입은 플래그를 건드리지 않음 — 재진입 자유.

## 하드 룰

- 이미지 필수, 영상은 모션이 정보일 때만.
- 스킵 가능 — Skip 버튼 항상 노출.
- "다시 보기" 1~2탭 내 접근.
- 튜토리얼 자체에 광고 절대 없음.
- 음성 없음.
- 일러스트가 실제 앱 UI를 충실히 반영 (가상 다른 앱 화면 금지).

## 결정 보류 (v1 빌드 시점에 정함)

image-to-video에 정확히 어느 API 엔드포인트를 쓸지 미확정 — Veo 2, Veo 3, Gemini 2.5 비디오 모달 등 시점별로 옵션이 달라진다. v1에서는 당시 가장 안정적이고 비용 합리적인 것을 선택하고, 결정 후 본 learning 파일을 갱신한다.
